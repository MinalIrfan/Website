Manual Deployment Steps

Step-by-step guide to deploy using XAMPP

1

Install XAMPP

Download and install XAMPP from apachefriends.org. Choose appropriate version for your OS (Windows, Linux, macOS).

Download: https://www.apachefriends.org
2

Prepare Website Files

Create your HTML, CSS, and JavaScript files. Organize them in a folder with proper structure.

mywebsite/
├── index.html
├── style.css
└── images/
3

Place in htdocs

Copy your website folder to XAMPP's htdocs directory (default: C:\xampp\htdocs).

C:\xampp\htdocs\mywebsite\
4

Start Apache Server

Open XAMPP Control Panel and start the Apache server. Ensure it's running on port 80.

XAMPP Control Panel → Apache → Start
5

Access Website

Open browser and navigate to localhost/your-folder-name to view your deployed website.

http://localhost/mywebsite

Deployment Methods

Different strategies for software deployment

Manual Deployment

Human operators execute deployment steps manually without automation.

Pros

  • Full control over process
  • Simple to understand
  • No automation setup needed

Cons

  • Time-consuming
  • Error-prone
  • Not scalable

Blue-Green

Two identical environments maintained for zero-downtime deployments.

Pros

  • Zero downtime
  • Instant rollback
  • Reduced risk

Cons

  • Double infrastructure
  • Higher costs
  • Complex setup

Rolling Deployment

Gradual replacement of instances while keeping application available.

Pros

  • No downtime
  • Resource efficient
  • Progressive validation

Cons

  • Version inconsistency
  • Longer deployment time
  • Complex monitoring
